Gestión de Repositorios con GitHub

Aprende a inicializar, clonar, actualizar y guardar repositorios en GitHub usando Git

Introducción a Git y GitHub

¿Qué es Git?

  • Sistema de control de versiones distribuido
  • Desarrollado por Linus Torvalds en 2005
  • Permite tracking de cambios en proyectos
  • Facilita la colaboración en equipo

¿Qué es GitHub?

  • Plataforma de alojamiento para repositorios Git
  • Servicio en la nube para proyectos
  • Funcionalidades sociales y de colaboración
  • Integración con muchas herramientas

Flujo de trabajo básico

  • Working Directory → Staging Area → Repository
  • git add → git commit → git push
Git es el sistema, GitHub es la plataforma que lo aloja

Configuración Inicial

Instalación de Git

  • Windows: Descargar desde git-scm.com
  • macOS: Usar Homebrew (brew install git)
  • Linux: Usar el gestor de paquetes

Configuración básica

# Configurar usuario y email
git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@ejemplo.com"

# Configurar editor preferido
git config --global core.editor "code --wait"

# Ver configuración
git config --list

Autenticación con GitHub

  • SSH keys (recomendado)
  • Personal Access Tokens
  • Git Credential Manager
La configuración correcta es esencial para un flujo de trabajo eficiente

Inicializar un Repositorio

Crear repositorio local

# Navegar al directorio del proyecto
cd mi-proyecto

# Inicializar repositorio Git
git init

# Ver estado del repositorio
git status

Estructura inicial

  • .git/ - Directorio de control de Git (oculto)
  • Working directory - Archivos de tu proyecto
  • Staging area - Área de preparación

Archivo .gitignore

# Ejemplo de .gitignore
node_modules/
*.log
.env
dist/
.DS_Store
git init crea un repositorio vacío o reinicializa uno existente

Clonar un Repositorio

Desde GitHub

# Clonar con HTTPS
git clone https://github.com/usuario/repositorio.git

# Clonar con SSH
git clone git@github.com:usuario/repositorio.git

# Clonar en directorio específico
git clone https://github.com/usuario/repositorio.git mi-carpeta

Desde otras fuentes

  • GitLab: Similar a GitHub
  • Bitbucket: Similar a GitHub
  • Repositorio local: git clone /ruta/al/repositorio

Después de clonar

# Ver ramas remotas
git branch -r

# Ver información del repositorio
git remote -v

# Ver historial de commits
git log --oneline
git clone crea una copia local de un repositorio remoto

Trabajar con Ramas

Gestión de ramas

# Ver ramas locales
git branch

# Crear nueva rama
git branch nueva-rama

# Cambiar a rama
git checkout nueva-rama
# O con el comando más moderno:
git switch nueva-rama

# Crear y cambiar a rama
git checkout -b nueva-rama

Ramas remotas

# Sincronizar ramas remotas
git fetch origin

# Crear rama local desde remota
git checkout -b mi-rama origin/mi-rama

# Publicar rama local en remoto
git push -u origin mi-rama
Las ramas permiten trabajar en características aisladas sin afectar el código principal

Hacer Commit de Cambios

Flujo de trabajo básico

# Ver cambios pendientes
git status

# Añadir archivos al staging
git add archivo.txt
git add . # Todos los archivos
git add *.js # Todos los archivos JS

# Hacer commit
git commit -m "Mensaje descriptivo del commit"

# Ver historial
git log --oneline

Buenas prácticas para commits

  • Commits atómicos (un cambio por commit)
  • Mensajes descriptivos y concisos
  • Usar el formato imperativo
  • Referenciar issues o tickets

Ejemplo de mensaje

  • feat: añadir funcionalidad de login
  • fix: corregir error en cálculo de total
  • docs: actualizar documentación de API
Un commit es un snapshot de los cambios en un punto específico del tiempo

Sincronizar con Remoto

Subir cambios (Push)

# Subir cambios al remoto
git push origin main

# Subir rama específica
git push origin mi-rama

# Subir y establecer upstream
git push -u origin mi-rama

# Forzar push (usar con cuidado)
git push -f origin main

Traer cambios (Pull)

# Descargar cambios del remoto
git pull origin main

# Equivalente a:
git fetch origin
git merge origin/main

Resolver conflictos

  • Git marca los archivos con conflictos
  • Editar manualmente los archivos conflictivos
  • Marcar como resuelto: git add archivo
  • Completar merge: git commit
git push sube tus commits locales, git pull trae commits remotos

Trabajar con Issues y PRs

GitHub Issues

  • Reportar bugs o errores
  • Sugerir nuevas características
  • Gestionar tareas del proyecto
  • Asignar responsables

Pull Requests

  • Proponer cambios al código
  • Revisión de código por pares
  • Integración continua
  • Discusión de implementación

Flujo de trabajo típico

# Crear rama para issue
git checkout -b fix/issue-123

# Hacer cambios y commits
git add .
git commit -m "fix: resolver issue #123"

# Subir rama
git push -u origin fix/issue-123

# Crear PR en GitHub
Los Issues y PRs facilitan la colaboración y el control de calidad

Comandos Avanzados Útiles

Historial y cambios

# Ver cambios en archivos
git diff

# Ver historial gráfico
git log --oneline --graph --all

# Ver cambios de un commit
git show abc1234

Gestión de cambios

# Deshacer cambios en archivo
git checkout -- archivo.txt

# Quitar archivos del staging
git reset HEAD archivo.txt

# Modificar último commit
git commit --amend

Stashing

# Guardar cambios temporales
git stash

# Ver stashes guardados
git stash list

# Aplicar último stash
git stash apply

# Aplicar stash específico
git stash apply stash@{1}
Estos comandos avanzados mejoran tu eficiencia con Git
