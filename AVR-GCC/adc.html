ADC en ATmega328P con AVR-GCC

Aprende a utilizar el Convertidor Analógico-Digital (ADC) con el microcontrolador ATmega328P programado en C usando AVR-GCC

Introducción al ADC

¿Qué es un ADC?

  • Convertidor Analógico-Digital (Analog-to-Digital Converter)
  • Convierte señales analógicas a valores digitales
  • Esencial para leer sensores (temperatura, luz, potenciómetros)
  • El ATmega328P tiene un ADC de 10 bits

Características del ADC

  • Resolución de 10 bits (1024 valores)
  • 6 canales multiplexados (ADC0 a ADC5)
  • Rango de voltaje: 0V a Vref (5V o referencia interna)
  • Conversión single-ended o diferencial
El ADC es una de las periferias más usadas en aplicaciones embebidas

Registros del ADC

Registros principales

  • ADMUX - ADC Multiplexer Selection Register
  • ADCSRA - ADC Control and Status Register A
  • ADCL/ADCH - ADC Data Register (Low/High)
  • DIDR0 - Digital Input Disable Register 0

Configuración de ADMUX

  • REFS1:0 - Selección de voltaje de referencia
  • ADLAR - Ajuste a izquierda del resultado
  • MUX3:0 - Selección de canal analógico
ADCSRA controla el habilitado, inicio de conversión y preescaler

Inicialización del ADC

Configuración básica

// Inicializar ADC
void adc_init() {
  // Referencia AVcc, canal ADC0
  ADMUX = (1 << REFS0);

  // Habilitar ADC, preescaler de 128
  // 16MHz/128 = 125kHz (dentro de 50-200kHz recomendado)
  ADCSRA = (1 << ADEN) | (1 << ADPS2) | (1 << ADPS1) | (1 << ADPS0);
}

Opciones de referencia

  • AREF - Voltaje externo
  • AVcc - Voltaje de alimentación (5V)
  • Interno - Referencia interna de 1.1V
Frecuencia de reloj del ADC debe estar entre 50kHz y 200kHz para máxima resolución

Lectura Analógica

Función de lectura ADC

// Leer valor ADC de un canal
uint16_t adc_read(uint8_t channel) {
  // Seleccionar canal
  ADMUX = (ADMUX & 0xF0) | (channel & 0x0F);

  // Iniciar conversión
  ADCSRA |= (1 << ADSC);

  // Esperar fin de conversión
  while (ADCSRA & (1 << ADSC));

  // Devolver resultado (leer ADCL primero)
  return ADC;
}

Canales ADC

  • ADC0 - Pin PC0
  • ADC1 - Pin PC1
  • ADC2 - Pin PC2
  • ADC3 - Pin PC3
  • ADC4 - Pin PC4 (SDA)
  • ADC5 - Pin PC5 (SCL)
Leer ADCL primero y luego ADCH para asegurar consistencia

Modo de Conversión Continua

Auto-Trigger y Interrupciones

// Inicializar ADC con auto-trigger
void adc_init_auto() {
  // Referencia AVcc, canal ADC0
  ADMUX = (1 << REFS0);

  // Habilitar ADC, auto-trigger, interrupción, preescaler 128
  ADCSRA = (1 << ADEN) | (1 << ADATE) | (1 << ADIE) |
           (1 << ADPS2) | (1 << ADPS1) | (1 << ADPS0);

  // Free Running mode
  ADCSRB = 0;

  // Habilitar interrupciones globales
  sei();

  // Iniciar primera conversión
  ADCSRA |= (1 << ADSC);
}

Rutina de interrupción

// ISR para conversión completa
ISR(ADC_vect) {
  uint16_t adc_value = ADC;
  // Procesar valor leído
}
El modo auto-trigger permite conversiones continuas sin intervención del CPU

Ejemplo Completo

Lectura de potenciómetro

#include <avr/io.h>
#include <util/delay.h>
#include <stdio.h>

// Inicializar ADC
void adc_init() {
  ADMUX = (1 << REFS0);
  ADCSRA = (1 << ADEN) | (1 << ADPS2) | (1 << ADPS1) | (1 << ADPS0);
}

// Leer canal ADC
uint16_t adc_read(uint8_t channel) {
  ADMUX = (ADMUX & 0xF0) | (channel & 0x0F);
  ADCSRA |= (1 << ADSC);
  while (ADCSRA & (1 << ADSC));
  return ADC;
}

int main(void) {
  adc_init();
  // Configurar UART para debug (opcional)

  while (1) {
    uint16_t valor = adc_read(0);
    // Convertir a voltaje (asumiendo Vref=5V)
    float voltaje = valor * (5.0 / 1023.0);
    _delay_ms(500);
  }
}
Compilar con: avr-gcc -mmcu=atmega328p -Os adc.c -o adc.elf
