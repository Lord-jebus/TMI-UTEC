AVR-GCC con Módulo RFID RC522

Programación en Arduino usando AVR-GCC y registros

Introducción a AVR-GCC

¿Qué es AVR-GCC?

  • Compilador de GNU para microcontroladores AVR
  • Permite programar en C/C++ para Arduino sin el IDE
  • Mayor control sobre el hardware y registros
  • Programas más eficientes y de menor tamaño

Ventajas para RFID

  • Control directo de los protocolos SPI
  • Manejo eficiente de interrupciones
  • Optimización de la comunicación con el RC522
Compilación: avr-gcc -mmcu=atmega328p -Os programa.c -o programa.elf

Configuración de Pines

Configuración de registros

En AVR-GCC, configuramos los pines directamente mediante registros:

// Configurar pin SS (Slave Select) como salida
#define SS_PIN PB2
#define DDR_SPI DDRB
#define PORT_SPI PORTB

// Inicializar pines SPI
void spi_init() {
  // Configurar MOSI, SCK y SS como salidas
  DDR_SPI |= (1 << PB3) | (1 << PB5) | (1 << SS_PIN);
  // Configurar MISO como entrada
  DDR_SPI &= ~(1 << PB4);
}
DDRx: Data Direction Register, PORTx: Puerto de salida

Comunicación SPI

Implementación de SPI con AVR-GCC

// Función para transferir datos via SPI
uint8_t spi_transfer(uint8_t data) {
  // Esperar hasta que el buffer de transmisión esté vacío
  while (!(SPSR & (1 << SPIF)));
  // Escribir el dato en el registro
  SPDR = data;
  // Esperar a que se complete la transferencia
  while (!(SPSR & (1 << SPIF)));
  // Devolver el dato recibido
  return SPDR;
}

El registro SPDR (SPI Data Register) se utiliza para enviar y recibir datos.

SPSR: SPI Status Register, SPDR: SPI Data Register

Inicialización del RC522

Función de inicialización

// Inicializar el módulo RC522
void rc522_init() {
  // Configurar el pin RST
  DDR_RC522 |= (1 << RST_PIN);
  PORT_RC522 &= ~(1 << RST_PIN);
  _delay_ms(1);
  PORT_RC522 |= (1 << RST_PIN);
  _delay_ms(50);

  // Inicializar comunicación SPI
  spi_init();

  // Realizar soft reset
  rc522_write(CommandReg, SoftReset_CMD);
  _delay_ms(50);
}
Se utiliza reset por hardware y software para inicializar correctamente

Lectura de Tarjetas RFID

Función de autenticación y lectura

// Leer bloque de datos de la tarjeta
int8_t rc522_read_card(uint8_t blockAddr, uint8_t* data) {
  uint8_t buffer[18];
  uint8_t size = sizeof(buffer);

  // Autenticar bloque
  if (rc522_auth(AntiCollision, blockAddr)) {
    // Leer bloque
    if (rc522_read(blockAddr, buffer, &size) == MI_OK) {
      memcpy(data, buffer, 16);
      return MI_OK;
    }
  }
  return MI_ERR;
}
Se requiere autenticación antes de leer cualquier bloque de datos

Optimizaciones con AVR-GCC

Ventajas de programar con AVR-GCC

  • Código más compacto y eficiente
  • Mayor control sobre los registros del microcontrolador
  • Posibilidad de optimizar funciones críticas en ensamblador
  • Uso de interrupciones para un mejor rendimiento

Comparación de tamaños

  • Sketch Arduino IDE: ~15-20KB
  • Compilado con AVR-GCC: ~8-12KB
  • Mejor manejo de memoria RAM
AVR-GCC permite flags de optimización como -Os para optimizar por tamaño
