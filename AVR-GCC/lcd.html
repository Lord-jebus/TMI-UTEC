Pantalla LCD Paralela e I2C con ATmega328P

Aprende a utilizar pantallas LCD 16x2 en modo paralelo y con adaptador I2C usando el microcontrolador ATmega328P programado en C con AVR-GCC

Introducción a LCD 16x2

Características

  • Pantalla de 16 caracteres × 2 líneas
  • Controlador HD44780 o compatible
  • Modo de 4 u 8 bits
  • Comunicación paralela o serial (I2C)
  • Bajo consumo de energía

Aplicaciones

  • Sistemas embebidos
  • Instrumentación
  • Interfaz de usuario
  • Proyectos educativos
La pantalla LCD 16x2 es el display más popular para proyectos con microcontroladores

Pines LCD Paralelo

Conexión básica

  • VSS - Tierra (GND)
  • VDD - Alimentación (+5V)
  • V0 - Contraste (potenciómetro)
  • RS - Register Select
  • RW - Read/Write (generalmente a GND)
  • E - Enable
  • D0-D7 - Bus de datos
  • A/K - Backlight +/-
En modo 4 bits se usan D4-D7, en modo 8 bits se usan D0-D7

Configuración Modo 4 Bits

Inicialización LCD

// Definición de pines
#define RS PD0
#define EN PD1
#define D4 PD4
#define D5 PD5
#define D6 PD6
#define D7 PD7

// Inicializar pines
void lcd_init_pins() {
  DDRD |= (1 << RS) | (1 << EN) | (1 << D4) |
           (1 << D5) | (1 << D6) | (1 << D7);
}
El modo 4 bits requiere menos pines pero es más lento que el modo 8 bits

Función de Envío de Comandos

Envío de nibbles (4 bits)

// Enviar nibble a la LCD
void lcd_send_nibble(uint8_t nibble) {
  if (nibble & 0x01) PORTD |= (1 << D4);
  else PORTD &= ~(1 << D4);
  if (nibble & 0x02) PORTD |= (1 << D5);
  else PORTD &= ~(1 << D5);
  if (nibble & 0x04) PORTD |= (1 << D6);
  else PORTD &= ~(1 << D6);
  if (nibble & 0x08) PORTD |= (1 << D7);
  else PORTD &= ~(1 << D7);

  // Pulso en Enable
  PORTD |= (1 << EN);
  _delay_us(1);
  PORTD &= ~(1 << EN);
  _delay_us(100);
}
El pulso en el pin E (Enable) es crucial para que la LCD capture los datos

Función de Inicialización LCD

Secuencia de inicialización

// Inicializar LCD en modo 4 bits
void lcd_init() {
  _delay_ms(50);
  PORTD &= ~(1 << RS); // Modo comando

  // Secuencia de inicialización
  lcd_send_nibble(0x03);
  _delay_ms(5);
  lcd_send_nibble(0x03);
  _delay_us(100);
  lcd_send_nibble(0x03);
  _delay_us(100);
  lcd_send_nibble(0x02); // Modo 4 bits

  // Configurar función: 4 bits, 2 líneas, font 5x8
  lcd_send_command(0x28);
  lcd_send_command(0x0C); // Display on, cursor off
  lcd_send_command(0x06); // Increment cursor
  lcd_send_command(0x01); // Clear display
  _delay_ms(2);
}
La secuencia de inicialización es crítica para el correcto funcionamiento

Función de Envío de Caracteres

Escribir en la LCD

// Enviar comando a la LCD
void lcd_send_command(uint8_t cmd) {
  PORTD &= ~(1 << RS); // Modo comando
  lcd_send_nibble(cmd >> 4);
  lcd_send_nibble(cmd & 0x0F);
}

// Enviar carácter a la LCD
void lcd_send_data(uint8_t data) {
  PORTD |= (1 << RS); // Modo datos
  lcd_send_nibble(data >> 4);
  lcd_send_nibble(data & 0x0F);
}

// Escribir cadena de texto
void lcd_write_string(char *str) {
  while (*str) {
    lcd_send_data(*str++);
  }
}
Los comandos se envían con RS=0, los datos con RS=1

Introducción a I2C

Protocolo I2C

  • Inter-Integrated Circuit
  • Comunicación serial sincrónica
  • Dos cables: SDA (datos) y SCL (reloj)
  • Protocolo maestro-esclavo
  • Múltiples dispositivos en el mismo bus

Ventajas para LCD

  • Solo 2 pines necesarios
  • Reducción de cableado
  • Múltiples dispositivos I2C
  • Más fácil de expandir
El protocolo I2C fue desarrollado por Philips en 1982

Adaptador I2C para LCD

Características del módulo

  • Basado en el chip PCF8574
  • Conversión de I2C a paralelo
  • Dirección I2C configurable (0x20-0x27)
  • Alimentación: 5V o 3.3V
  • Potenciómetro integrado para contraste

Conexión

  • GND - Tierra
  • VCC - Alimentación 5V
  • SDA - Datos I2C
  • SCL - Reloj I2C
El módulo I2C simplifica enormemente la conexión de la LCD

Configuración I2C en ATmega328P

Inicialización I2C (TWI)

// Inicializar I2C a 100kHz
void i2c_init() {
  // Establecer frecuencia: 16MHz/(16+2*TWBR*1)
  TWSR = 0; // Preescaler 1
  TWBR = ((F_CPU / 100000) - 16) / 2;
  TWCR = (1 << TWEN); // Habilitar TWI
}

// Iniciar condición de start
void i2c_start() {
  TWCR = (1 << TWINT) | (1 << TWSTA) | (1 << TWEN);
  while (!(TWCR & (1 << TWINT)));
}
El módulo TWI (Two-Wire Interface) del ATmega328P maneja la comunicación I2C

Función de Escritura I2C

Envío de datos por I2C

// Enviar byte por I2C
void i2c_write(uint8_t data) {
  TWDR = data;
  TWCR = (1 << TWINT) | (1 << TWEN);
  while (!(TWCR & (1 << TWINT)));
}

// Detener comunicación I2C
void i2c_stop() {
  TWCR = (1 << TWINT) | (1 << TWSTO) | (1 << TWEN);
}

Escritura al módulo I2C

// Enviar comando a LCD por I2C
void lcd_i2c_write(uint8_t data, uint8_t mode) {
  uint8_t high = data & 0xF0;
  uint8_t low = (data << 4) & 0xF0;
  uint8_t data_arr[4];
  data_arr[0] = high | mode | LCD_BACKLIGHT | EN;
  data_arr[1] = high | mode | LCD_BACKLIGHT;
  data_arr[2] = low | mode | LCD_BACKLIGHT | EN;
  data_arr[3] = low | mode | LCD_BACKLIGHT;

  i2c_start();
  i2c_write(LCD_ADDR << 1);
  for (uint8_t i = 0; i < 4; i++) {
    i2c_write(data_arr[i]);
  }
  i2c_stop();
}
La escritura I2C requiere el envío de la dirección del dispositivo y los datos

Inicialización LCD con I2C

Secuencia de inicialización I2C

// Inicializar LCD con I2C
void lcd_i2c_init() {
  _delay_ms(50);
  // Secuencia de inicialización
  lcd_i2c_write(0x03, LCD_COMMAND);
  _delay_ms(5);
  lcd_i2c_write(0x03, LCD_COMMAND);
  _delay_us(100);
  lcd_i2c_write(0x03, LCD_COMMAND);
  _delay_us(100);
  lcd_i2c_write(0x02, LCD_COMMAND);

  // Configurar función
  lcd_i2c_write(0x28, LCD_COMMAND);
  lcd_i2c_write(0x0C, LCD_COMMAND);
  lcd_i2c_write(0x06, LCD_COMMAND);
  lcd_i2c_write(0x01, LCD_COMMAND);
  _delay_ms(2);
}
La secuencia de inicialización es similar al modo paralelo pero usando I2C

Ejemplo Completo con I2C

Programa completo

#include <avr/io.h>
#include <util/delay.h>

#define LCD_ADDR 0x27
#define LCD_COMMAND 0
#define LCD_DATA 1
#define LCD_BACKLIGHT 0x08
#define EN 0x04

// Prototipos de funciones
void i2c_init();
void i2c_start();
void i2c_write(uint8_t data);
void i2c_stop();
void lcd_i2c_write(uint8_t data, uint8_t mode);
void lcd_i2c_init();
void lcd_i2c_write_string(char *str);

int main() {
  i2c_init();
  lcd_i2c_init();

  while(1) {
    lcd_i2c_write(0x80, LCD_COMMAND); // Línea 1
    lcd_i2c_write_string("Hola Mundo!");
    lcd_i2c_write(0xC0, LCD_COMMAND); // Línea 2
    lcd_i2c_write_string("ATmega328P+I2C");
    _delay_ms(1000);
  }
}
Compilar con: avr-gcc -mmcu=atmega328p -Os lcd_i2c.c -o lcd_i2c.elf
