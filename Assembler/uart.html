Comunicación UART en ATmega328P

Aprende a utilizar el módulo UART del ATmega328P programado en ensamblador para comunicación serial

Introducción a UART

¿Qué es UART?

  • Universal Asynchronous Receiver/Transmitter
  • Protocolo de comunicación serial asíncrono
  • Comunicación punto a punto full-duplex
  • No requiere reloj compartido
  • Usa 2 cables: TX (transmisión) y RX (recepción)

Características en ATmega328P

  • Módulo USART (Universal Synchronous/Asynchronous Receiver/Transmitter)
  • Soporta comunicación asíncrona y síncrona
  • Baud rates desde 2400 hasta 2Mbps (a 16MHz)
  • Registros separados para recepción y transmisión
  • Interrupciones para recepción y transmisión
UART es uno de los protocolos de comunicación más antiguos y ampliamente utilizados

Registros UART del ATmega328P

Registros Principales

  • UDR0 (USART Data Register): Datos de entrada/salida
  • UCSR0A (USART Control and Status Register A): Estado y control
  • UCSR0B (USART Control and Status Register B): Interrupciones y habilitación
  • UCSR0C (USART Control and Status Register C): Formato
  • UBRR0L y UBRR0H (USART Baud Rate Registers): Velocidad

Bits Importantes

  • RXEN0: Habilitar recepción
  • TXEN0: Habilitar transmisión
  • RXCIE0: Habilitar interrupción por recepción
  • TXCIE0: Habilitar interrupción por transmisión
  • UDRE0: Buffer de datos vacío (listo para transmitir)
  • RXC0: Recepción completada
La correcta configuración de los registros es esencial para el funcionamiento del UART

Configuración de Baud Rate

Cálculo de Baud Rate

  • Fórmula: UBRR = (F_CPU / (16 * BAUD)) - 1
  • Para 16MHz y 9600 baud: UBRR = (16000000 / (16 * 9600)) - 1 = 103
  • Para 16MHz y 115200 baud: UBRR = (16000000 / (16 * 115200)) - 1 = 8
  • El registro UBRR0 es de 16 bits (UBRR0H y UBRR0L)

Configuración en Ensamblador

; Configurar para 9600 baudios a 16MHz
ldi r16, HIGH(103) ; 103 = (16000000/(16*9600))-1
sts UBRR0H, r16
ldi r16, LOW(103)
sts UBRR0L, r16

Baud Rates Comunes

  • 2400, 4800, 9600, 19200, 38400
  • 57600, 115200 (máxima velocidad estable)
  • 250000, 500000 (requieren cristal preciso)
La velocidad de comunicación debe coincidir en ambos dispositivos

Configuración de Formato

Opciones de Formato

  • Bits de datos: 5, 6, 7, 8 o 9 bits
  • Bits de parada: 1 o 2 bits
  • Paridad: Ninguna, par o impar
  • Modo de operación: Asíncrono o síncrono

Configuración de UCSR0C

  • UMSEL01:0: Modo (00=Asíncrono, 01=Síncrono)
  • UPM01:0: Paridad (00=Ninguna, 10=Par, 11=Impar)
  • USBS0: Bits de parada (0=1bit, 1=2bits)
  • UCSZ01:0: Tamaño de datos (11=8bits con UCSZ02)
  • UCSZ02: Bit adicional para 9 bits (en UCSR0B)

Ejemplo de Configuración

; Configurar para 8 bits de datos, 1 bit de parada, sin paridad
ldi r16, (1<<UCSZ01)|(1<<UCSZ00) ; 8 bits
sts UCSR0C, r16
El formato 8N1 (8 bits, no paridad, 1 bit de parada) es el más común

Inicialización Completa del UART

Secuencia de Inicialización

  1. Configurar baud rate (UBRR0H y UBRR0L)
  2. Configurar formato (UCSR0C)
  3. Habilitar transmisor y/o receptor (UCSR0B)
  4. Opcional: Habilitar interrupciones

Función de Inicialización

; Inicializar UART para 9600 8N1
uart_init:
; Configurar baud rate (9600 a 16MHz)
ldi r16, HIGH(103)
sts UBRR0H, r16
ldi r16, LOW(103)
sts UBRR0L, r16
; Configurar formato (8 bits, 1 stop, sin paridad)
ldi r16, (1<<UCSZ01)|(1<<UCSZ00)
sts UCSR0C, r16
; Habilitar transmisor y receptor
ldi r16, (1<<RXEN0)|(1<<TXEN0)
sts UCSR0B, r16
ret
La inicialización debe realizarse antes de cualquier operación de comunicación

Transmisión de Datos

Transmisión por Polling

  • Esperar a que el buffer de transmisión esté vacío (UDRE0)
  • Escribir dato en UDR0
  • El hardware se encarga de la transmisión serial

Función de Transmisión

; Transmitir carácter por UART (polling)
; Entrada: r16 - carácter a transmitir
uart_tx:
; Esperar a que el buffer de transmisión esté vacío
uart_tx_wait:
  lds r17, UCSR0A
  sbrs r17, UDRE0 ; Saltar si UDRE0 está set
  rjmp uart_tx_wait
; Escribir dato en buffer de transmisión
  sts UDR0, r16
  ret

Transmisión por Interrupciones

  • Habilitar interrupción UDRE0 (buffer vacío)
  • Escribir handler de interrupción
  • Más eficiente pero más complejo
La transmisión por polling es simple pero bloqueante

Recepción de Datos

Recepción por Polling

  • Esperar a que se reciba un dato (RXC0)
  • Leer dato de UDR0
  • Procesar el dato recibido

Función de Recepción

; Recibir carácter por UART (polling)
; Salida: r16 - carácter recibido
uart_rx:
; Esperar a que se reciba un dato
uart_rx_wait:
  lds r17, UCSR0A
  sbrs r17, RXC0 ; Saltar si RXC0 está set
  rjmp uart_rx_wait
; Leer dato recibido
  lds r16, UDR0
  ret

Recepción por Interrupciones

  • Habilitar interrupción RXC0 (dato recibido)
  • Escribir handler de interrupción
  • Permite procesamiento no bloqueante

Handler de Interrupción

; Handler de interrupción por recepción
USART_RX_vect:
  push r16
  in r16, SREG
  push r16
  lds r16, UDR0 ; Leer dato recibido
  ; Procesar dato (almacenar en buffer, etc.)
  pop r16
  out SREG, r16
  pop r16
  reti
La recepción por interrupciones es más eficiente para aplicaciones en tiempo real

Transmisión de Cadenas

Transmisión de Mensajes

  • Transmitir caracteres individualmente en un bucle
  • Usar tablas de strings en memoria de programa
  • Terminar strings con carácter nulo (0x00)
  • Incluir caracteres especiales (CR, LF)

Función de Transmisión de String

; Transmitir string desde memoria de programa
; Entrada: Z - apuntador al string (terminado en 0)
uart_print:
  lpm r16, Z+ ; Cargar carácter desde flash
  cpi r16, 0 ; ¿Es el terminador nulo?
  breq uart_print_end
  rcall uart_tx ; Transmitir carácter
  rjmp uart_print
uart_print_end:
  ret

; String en memoria de programa
mensaje: .db "Hola UART!", 0x0D, 0x0A, 0
; Uso:
  ldi ZL, LOW(mensaje<<1)
  ldi ZH, HIGH(mensaje<<1)
  rcall uart_print

Caracteres Especiales

  • CR (Carriage Return): 0x0D
  • LF (Line Feed): 0x0A
  • CR+LF: Estándar para nueva línea
  • NULL: 0x00 (terminador de string)
La transmisión de strings completos es útil para mensajes de depuración

Ejemplo Completo: Eco por UART

Programa de Eco

  • Lee caracteres recibidos por UART
  • Los retransmite inmediatamente
  • Útil para probar la comunicación
  • Demo clásica de funcionamiento UART

Código Completo

; Programa de eco por UART
.include "m328Pdef.inc"

; Punto de entrada
main:
  rcall uart_init ; Inicializar UART
  ldi ZL, LOW(mensaje_inicio<<1)
  ldi ZH, HIGH(mensaje_inicio<<1)
  rcall uart_print ; Mensaje inicial
loop:
  rcall uart_rx ; Leer carácter
  rcall uart_tx ; Transmitir carácter (eco)
  rjmp loop ; Repetir

; Mensaje inicial
mensaje_inicio: .db "Eco UART activo", 0x0D, 0x0A, 0

Funciones de Inicialización

; (Aquí irían las funciones uart_init, uart_tx, uart_rx, uart_print)
El programa de eco es el "Hola Mundo" de la comunicación UART

Depuración y Solución de Problemas

Problemas Comunes

  • Baud rate incorrecto
  • Configuración de formato no coincidente
  • Conexión TX-RX cruzada incorrectamente
  • Niveles de voltaje incorrectos (5V vs 3.3V)
  • Problemas de timing con cristal no preciso

Técnicas de Depuración

  • Usar LED para indicar actividad
  • Transmitir mensajes de estado
  • Verificar registros con debugger
  • Probar con diferentes baud rates
  • Usar osciloscopio para verificar señal

Verificación de Configuración

; Función para verificar configuración UART
uart_check_config:
  lds r16, UCSR0B
  sbrs r16, RXEN0
  rcall error_rx_disabled
  sbrs r16, TXEN0
  rcall error_tx_disabled
  ret
Una buena estrategia de depuración ahorra tiempo en el desarrollo
